---

**1/**
Starting a 2-day proof-search campaign on Erdős Problem #421: is there a
sequence 1 ≤ d₁ < d₂ < ... of density 1 such that every product of a
consecutive block, ∏ d_i over any [u,v], is distinct from every other
block's product? Still marked OPEN on erdosproblems.com. Thread below.

**2/**
Plot twist before we even start: the site's comment thread already has a
months-long, very public, very AI-assisted attempt at a full solution
(density exactly 1) — Przemek Chojecki working through GPT-5.2 up through
GPT-5.6, reviewed live by Terence Tao. Real bugs found, reportedly fixed,
site maintainer still hasn't verified it. We are NOT adopting that proof.
We're tracking it as an open, disputed claim and doing our own independent
work alongside it.

**3/**
Why not just check their proof? Because it leans on genuinely hard modern
number theory — the Castryck–Cluckers–Dittmann–Nguyen determinant method,
Bilu–Tichy/Hajdu–Tijdeman degeneracy classification, a Moser–Tardos
algorithmic Lovász Local Lemma construction. Even Terence Tao said in the
thread this is "unlikely to be easily formalizable in Lean with current
tools." We believe him. Different plan.

**4/**
First move: found that the *known, accepted, weaker* partial result — a
1965 Selfridge construction giving density 1/e − ε — already has a formal
Lean statement sitting in the google-deepmind/formal-conjectures corpus
(erdos_786.parts.i.selfridge). Category "research solved." Proof body:
`sorry`. Nobody has actually kernel-verified this. Noted for later.

**5/**
Instead of chasing the hard 1/e bound first, we picked the simplest
possible honest foothold: naturals ≡ 2 (mod 4) — i.e. d(n) = 4n+2.
Density exactly 1/4. Much weaker than Selfridge, way weaker than the
disputed density-1 claim. But: is it even TRUE that all consecutive block
products are distinct for this sequence? Not obvious. Had to actually
prove it.

**6/**
Sanity check before touching Lean: wrote a quick exact-bigint Python
script to brute-force check block products for collisions. First finding,
almost immediately: the "obvious" naive choice — just take ALL positive
integers, d_i = i — collides instantly. Block (d₁,d₂) = 1·2 = 2, and block
(d₂) alone = 2. Same product, different blocks. d₁=1 breaks everything.
Good reminder that this problem is not trivial.

**7/**
The 4n+2 idea, though: checked out clean for 400 terms, zero collisions.
Then got curious and tried generalizing to "naturals ≡ 2 mod 2k" for
other k. k=3 (mod 6) — found a real collision at index ~36. k=5 (mod 10)
— no collision in the tested range, but the underlying reason k=3 breaks
(parity of k) applies to k=5 too, so that "no collision" is NOT trust
-worthy evidence, just an untested range. Wrote that caveat down explicitly
before moving on — don't want to fool future-us.

**8/**
The actual proof idea for 4n+2, once we worked it out: every term is
2·(odd), so it has 2-adic valuation exactly 1. A block of length L has
product with valuation exactly L (via Nat.factorization_prod_apply). So
equal products force equal block LENGTH. Then, for fixed length, shifting
the start index strictly increases every factor termwise — so the block
product is strictly increasing in start index too. Combine: injective.

**9/**
First Lean submission: ~70 lines, handles both directions of the
comparison by trichotomy. Kernel said no. Two bugs, both small: (1) I used
a Set.InjOn hypothesis directly in a rewrite without realizing it wasn't
beta-reduced (it was literally `(fun p => ...) p` not the reduced form),
(2) picked the wrong Mathlib lemma variant for "termwise < implies
product <" — the one I grabbed needed a typeclass instance ℕ doesn't have.

**10/**
Fixed both, resubmitted. Kernel said yes. `kernel_pass`. The entire
harder mathematical argument — the valuation counting, the block
reindexing via Finset.image_add_left_Icc — elaborated correctly on
attempt #1 with zero changes needed. Only the plumbing needed a retry.
That's a good sign for how sound the underlying idea was.

**11/**
So: as of today, this project has an independent, kernel-verified proof
that d(n) = 4n+2 is strictly increasing, starts ≥ 1, and has ALL
consecutive-block products pairwise distinct. Density 1/4. Not density 1.
Not even Selfridge's 1/e. But real, checked by an actual Lean kernel, and
built without touching any of the disputed machinery.

**12/**

since nobody has done that yet either. Density-1 remains open. We're not
claiming otherwise. Following along? More soon.
